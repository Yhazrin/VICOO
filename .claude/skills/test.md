# 测试工程

创建全面的测试用例，确保代码质量。

## 用法

```bash
/test [target]
```

## 测试金字塔

```
       /\
      /  \     E2E 测试 (少量)
     /----\
    /      \   集成测试 (中等)
   /--------\  单元测试 (大量)
```

## 测试类型

### 1. 单元测试
- 测试单个函数/组件
- 快速执行
- Mock 外部依赖

### 2. 集成测试
- 测试多个组件协作
- 使用真实数据库/服务
- 较慢

### 3. E2E 测试
- 模拟用户行为
- 完整流程
- 最慢

## 测试驱动开发 (TDD)

1. **Red**: 写一个失败的测试
2. **Green**: 写最少的代码让测试通过
3. **Refactor**: 重构代码

## 测试覆盖

- 正常路径
- 边界条件
- 错误处理
- 并发场景

## 常用框架

- 前端: Vitest, Jest, React Testing Library
- 后端: Jest, Mocha, Supertest
- E2E: Playwright, Cypress
