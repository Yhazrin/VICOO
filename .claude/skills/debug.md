# 调试助手

帮助定位和修复代码中的 bug。

## 用法

```bash
/debug [issue-description]
```

## 调试流程

1. **复现问题**: 确定在什么情况下会出现 bug
2. **收集信息**: 查看错误消息、堆栈跟踪、日志
3. **定位根因**: 找出问题的根本原因
4. **修复问题**: 应用修复
5. **验证修复**: 确认问题已解决

## 常见调试技术

### 1. 日志分析
- 添加 console.log 或使用日志库
- 关注错误堆栈
- 检查请求/响应

### 2. 断点调试
- 使用 IDE 断点
- 使用 browser DevTools
- 使用 Node.js inspector

### 3. 隔离问题
- 最小化复现步骤
- 二分查找法
- 逐步注释代码

### 4. 类型检查
- 使用 TypeScript
- 使用 ESLint
- 检查类型推断

## 常见错误模式

- 空指针异常: 使用可选链 (?.) 或默认值
- 异步问题: 检查 await/then 使用
- 内存泄漏: 检查事件监听器、计时器
- 竞态条件: 使用锁或队列

## 输出格式

发现 bug 时：
```
## Bug [编号]

**位置**: 文件路径
**现象**: 描述问题现象
**原因**: 分析可能的原因
**修复**: 建议的修复方案
```
