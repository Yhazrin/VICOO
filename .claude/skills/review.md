# 代码审查

帮助进行有效的代码审查。

## 用法

```bash
/review [file-path]
```

## 审查重点

### 1. 正确性
- 代码是否正确实现功能？
- 边界条件是否处理？
- 错误是否正确处理？

### 2. 安全性
- 是否有安全漏洞？
- 输入是否验证？
- 敏感数据是否保护？

### 3. 性能
- 是否有性能问题？
- 数据库查询是否优化？
- 是否有不必要的计算？

### 4. 可读性
- 命名是否清晰？
- 是否有注释？
- 代码是否简洁？

### 5. 可测试性
- 是否容易测试？
- 是否有单元测试？

## 审查清单

- [ ] 功能正确
- [ ] 边界条件
- [ ] 错误处理
- [ ] 安全检查
- [ ] 性能考虑
- [ ] 代码风格
- [ ] 命名规范
- [ ] 注释充分
- [ ] 有测试
- [ ] 文档更新

## 审查评论

### 使用建设性语言
```
❌ "这代码写得不对"
✅ "这个实现可以简化..."
```

### 提供解决方案
```
❌ "这里有内存泄漏"
✅ "建议使用 useEffect 的清理函数..."
```

### 区分重要程度
- **必须修复**: 阻塞性问题
- **建议修改**: 改进建议
- **可选**: 偏好问题

## 常见问题

1. **过度工程**: 过度设计
2. **复制粘贴**: 代码重复
3. **魔法数字**: 硬编码值
4. **嵌套过深**: 条件复杂
5. **函数过长**: 职责过多
