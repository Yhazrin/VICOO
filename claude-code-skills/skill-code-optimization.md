# Vicoo 代码优化 Skill

## 目标
持续发现系统中的优化点并自我更新改进。

## 优化检查清单

每次处理任务时，必须自动检查以下优化点：

### 1. 代码质量优化
- [ ] 检查并移除不必要的 console.log/console.error（生产环境）
- [ ] 检查重复代码，提取公共函数
- [ ] 检查未使用的变量和导入
- [ ] 检查类型安全，避免使用 any

### 2. React/前端优化
- [ ] 检查 useEffect 依赖数组是否完整
- [ ] 检查是否使用了 React.FC（建议使用更现代的写法）
- [ ] 检查是否有不必要的重新渲染（使用 useMemo/useCallback）
- [ ] 检查图片和静态资源是否被正确优化

### 3. 后端/性能优化
- [ ] 检查数据库查询是否有 N+1 问题
- [ ] 检查是否有不必要的循环和重复计算
- [ ] 检查 API 是否有适当的错误处理
- [ ] 检查是否有内存泄漏风险

### 4. 安全优化
- [ ] 检查敏感信息是否暴露在日志中
- [ ] 检查输入验证是否充分
- [ ] 检查权限控制是否正确

### 5. TypeScript 最佳实践
- [ ] 检查是否使用了 strict 模式
- [ ] 检查类型定义是否完整
- [ ] 检查是否有隐式的 any

## 自我反思问题

每完成一个任务后，必须问自己：

1. 这段代码是否有更优雅的实现方式？
2. 是否有性能可以提升的地方？
3. 是否有重复的代码可以抽象？
4. 是否有潜在的 bug 风险？
5. 代码是否易于维护和扩展？

## 持续改进流程

1. **任务开始前**: 快速浏览相关代码，寻找优化点
2. **编写代码时**: 遵循最佳实践，避免引入新问题
3. **任务完成后**: 回顾代码，识别可改进之处
4. **定期**: 全面审查项目，识别系统性问题

## 优化优先级

高优先级：
- 安全性问题
- 明显性能问题
- 严重 bug

中优先级：
- 代码重复
- 类型安全问题
- 错误处理缺失

低优先级：
- 代码风格
- 小优化
- 注释改进

## 输出格式

当发现优化点时，使用以下格式记录：

```
## 优化点 [编号]

**位置**: 文件路径
**问题**: 描述问题
**建议**: 改进建议
**优先级**: P0/P1/P2
```
